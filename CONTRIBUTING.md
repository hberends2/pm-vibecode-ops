# Contributing to PM Vibe Code Operations

Thank you for your interest in contributing! This project aims to help non-technical Product Managers build production-quality software with AI assistance.

## Ways to Contribute

### Share Your Experience

The most valuable contributions come from real usage:

- **Add example scenarios** to [EXAMPLES.md](EXAMPLES.md)
- **Improve documentation** based on what confused you
- **Add FAQ entries** for questions you had
- **Expand the glossary** with terms that needed explanation

### Report Issues

Found a problem? Open an issue with:

- What you were trying to do
- What happened instead
- Your environment (OS, Claude Code version, Node version)
- Any error messages

### Suggest Improvements

Ideas for making this better? Open an issue describing:

- The problem or limitation you encountered
- Your proposed solution
- How it would help other PMs

## Contribution Guidelines

### Documentation Changes

- Keep language accessible to non-technical readers
- Test any command examples before submitting
- Update the table of contents when adding sections
- Cross-link to related documentation

### Command/Agent Changes

- Test changes against a real codebase
- Verify MCP integrations work
- Update CLAUDE.md if behavior changes
- Follow existing patterns and formatting

### Code of Conduct

- Be respectful and constructive
- Remember our audience is non-technical PMs
- Assume good intent
- Help others learn

## Pull Request Process

1. Fork the repository
2. Create a feature branch: `feature/your-improvement`
3. Make your changes
4. Test thoroughly
5. Submit a PR with:
   - Clear description of changes
   - Why the change improves the project
   - Any testing you performed
